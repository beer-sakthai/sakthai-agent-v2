# Hope — Solution Engineering
> Stage 2 of 6 · Sak Family cycle · หวัง (Wǎng)

> **Mantra** — *Turn the vision into a plan worth defending.*

## Charge

See [SOUL.md](./SOUL.md) for the charge system.

- **Gain charge** — entering with a clear Dream restores energy.
- **Spend charge** — designing, prototyping, and writing code drain energy.
- Hope is the **most ambitious** stage. Enter it well-charged.

## Goal

Translate the Dream vision into a concrete, actionable plan using **PTCF**:

- **P**roblem — what exactly must be solved.
- **T**echnique — the chosen approach, and why.
- **C**onsiderations — constraints, trade-offs, alternatives.
- **F**eedback — lessons recorded for future cycles.

## What this stage means

Hope converts ambition into structure. Decompose the goal, choose an approach,
defend it against alternatives, and begin building. A good Hope plan is
**disprovable** — stated clearly enough that Care can audit it.

## Do

- Capture each meaningful decision as a fact: `sakthai learn "<decision>" --kind note --tag decision`.
- Sketch the smallest version that proves the approach.

## Exit criteria

The plan is written, the key decisions are in memory, and a first implementation
exists to hand to Care. Advance: `sakthai cycle next` → **Care**.
